# Velox Detailed Reference

API reference for schema definitions, annotations, and generated code patterns. Only read sections relevant to the current task.

## Schema Definition Pattern

```go
import (
    "github.com/syssam/velox"
    "github.com/syssam/velox/schema/field"
    "github.com/syssam/velox/schema/edge"
    "github.com/syssam/velox/schema/index"
    "github.com/syssam/velox/schema/mixin"
)

type User struct{ velox.Schema }

func (User) Mixin() []velox.Mixin   { return []velox.Mixin{mixin.ID{}, mixin.Time{}} }
func (User) Fields() []velox.Field  { return []velox.Field{field.String("email").Unique()} }
func (User) Edges() []velox.Edge    { return []velox.Edge{edge.To("posts", Post.Type)} } // O2M default
func (User) Indexes() []velox.Index { return []velox.Index{index.Fields("email", "status")} }
```

Edge types: `To()` = O2M default, `From()` = belongs-to, `.Unique()` = O2O, `.Through()` = M2M.

## Edge Direction Convention

Velox follows Ent's convention: **the side that uses `edge.To` owns the relation.** The other side uses `edge.From(...).Ref(...)` as a back-reference. FK placement is decided by `Unique()`, not by which side is called the "parent."

Two perspectives are both idiomatic — pick the one that reads naturally for the domain.

### Perspective A — Parent owns the collection ("User has Posts")

```go
// schema/user.go
func (User) Edges() []velox.Edge {
    return []velox.Edge{
        edge.To("posts", Post.Type),
    }
}

// schema/post.go
func (Post) Edges() []velox.Edge {
    return []velox.Edge{
        edge.From("author", User.Type).
            Ref("posts").
            Unique().
            Required(),
    }
}
```

### Perspective B — Child owns its foreign key ("Post belongs to User")

```go
// schema/post.go
func (Post) Edges() []velox.Edge {
    return []velox.Edge{
        edge.To("author", User.Type).
            Field("author_id").
            Unique(),
    }
}

// schema/user.go
func (User) Edges() []velox.Edge {
    return []velox.Edge{
        edge.From("posts", Post.Type).Ref("author"),
    }
}
```

Both produce the same SQL and FK placement (on `Post`, column `author_id` in perspective B; default-named `user_posts` in perspective A). Use `edge.Field()` only on the `edge.To` side — that's the side that owns the relation. The choice is stylistic: A emphasizes the parent's collection; B emphasizes the child's foreign key and is useful when the FK is already a first-class field on the child.

### One-to-One (User ↔ Profile)

```go
// owner side
edge.To("profile", Profile.Type).Unique()

// back-reference
edge.From("user", User.Type).Ref("profile").Unique()
```

### Many-to-Many (User ↔ Group)

```go
// owner side
edge.To("groups", Group.Type)

// back-reference
edge.From("users", User.Type).Ref("groups")
```

### Self-Referencing

```go
// Tree: parent/children on the same type
edge.To("children", Node.Type).From("parent").Unique()

// Symmetric: bidirectional, no owner/inverse distinction
edge.To("friends", User.Type)
```

### Required edges

Use `Required()` only on one side. Velox rejects `Required()` on both ends of the same edge.

## Entity-Level GraphQL Annotations

```go
func (User) Annotations() []velox.Annotation {
    return []velox.Annotation{
        graphql.RelayConnection(),
        graphql.QueryField(),
        graphql.Type("Member"),                 // Custom GraphQL type name
        graphql.Mutations(                      // Opt-in (NOT generated by default)
            graphql.MutationCreate(),
            graphql.MutationUpdate(),
        ),
        graphql.Skip(graphql.SkipWhereInput),
        graphql.Directives(graphql.Directive{Name: "deprecated", Args: map[string]any{"reason": "Use Member"}}),
    }
}
```

Mutation shorthand: `graphql.Mutations()` with no args = Create + Update (Ent default).

## WhereInput Filtering (Whitelist Model)

Fields are NOT filterable by default. Opt-in per field/edge:

```go
// Field-level
field.String("email").Annotations(graphql.WhereInput())
field.String("role").Annotations(graphql.WhereOps(graphql.OpsEquality))

// Edge-level
edge.To("posts", Post.Type).Annotations(graphql.WhereInput())

// Entity-level bulk opt-in
graphql.WhereInputFields("email", "status", "created_at")
graphql.WhereInputEdges("posts")

// Legacy Ent-compat mode (all fields filterable)
gen.WithFeatures(gen.FeatureWhereInputAll)
```

## Field-Level Annotations

```go
graphql.OrderField("EMAIL")              // Custom order field name
graphql.Skip(graphql.SkipAll)            // Exclude from GraphQL
sqlschema.ColumnType("JSONB")            // Custom SQL column type
sqlschema.Check("length(data) > 0")      // CHECK constraint
sqlschema.OnDelete(sqlschema.Cascade)     // FK cascade (edge-level)
sqlschema.IndexType("GIN")               // Index type (index-level)
```

Enums auto-uppercase for GraphQL: DB `"big"` -> GraphQL `BIG`. Override with `graphql.EnumValues()`.

## Resolver Annotations

```go
graphql.Resolvers(
    graphql.Map("priceListItem(priceListId: ID!)", "PriceListItem!").WithComment("..."),
    graphql.Map("glAccount", "PublicGlAccount!"),
    graphql.Map("approver", "PublicUser"),
)
```

## GraphQL Input Validation

Uses go-playground/validator struct tags on generated input types:

```go
field.String("email").Annotations(
    graphql.CreateInputValidate("required,email"),
    graphql.UpdateInputValidate("omitempty,email"),
)
// Or convenience shorthand for both:
graphql.MutationInputValidate("required,email", "omitempty,email")
```

## Field Nullability

| Configuration | GraphQL Input | DB Column | Go Type | ORM Default |
|---------------|---------------|-----------|---------|-------------|
| (default) | `String!` | NOT NULL | `string` | Must be set |
| `Optional()` (standard type) | `String` | NOT NULL | `string` | Go zero value (ORM only) |
| `Optional()` (enum/custom) | ERROR | - | - | Requires `Default()` |
| `Optional().Default("x")` | `String` | NOT NULL + DEFAULT | `string` | `"x"` |
| `Nillable()` | `String` | NULL | `*string` | `nil` |
| `Optional().Nillable()` | `String` | NULL | `*string` | `nil` |

IMPORTANT: `Optional()` does NOT add DB DEFAULT. For migration safety use `.Default()`.

## Defaults Precedence (for migrations)

1. `sqlschema.DefaultExpr(expr)` — SQL expression (highest)
2. `sqlschema.Default(v)` — SQL literal
3. `Default(literal)` — fallback

`Default(func)` (e.g., `time.Now`, `uuid.New`) only runs in Go, has no SQL effect.
`UpdateDefault(func)` runs on every update. Skip with `SkipDefaults()` or `SkipDefault(field)`.

## FeatureAutoDefault

Auto-adds DB DEFAULT for all NOT NULL fields. Enables safe `ALTER TABLE ADD COLUMN`.

```go
gen.WithFeatures(gen.FeatureAutoDefault)
```

Works for: String(`''`), Int/Uint/Float(`0`), Bool(`false`).
Does NOT auto-default: Enum, JSON, Time, UUID, Bytes, Nillable fields.

## ORM Validators (opt-in)

Requires `gen.FeatureValidator`. Methods: `NotEmpty()`, `MinLen(n)`, `MaxLen(n)`, `Match(re)`, `Positive()`, `NonNegative()`, `Min(n)`, `Max(n)`, `Range(min, max)`.

## Schema Split Modes

| Mode | Output |
|------|--------|
| `SchemaSplitNone` (default) | Single `schema.graphql` |
| `SchemaSplitByCategory` | `schema/{types,inputs,connections,scalars}.graphql` |
| `SchemaSplitPerEntity` | `velox.graphql` + `velox_{entity}.graphql` per entity |

`SchemaSplitPerEntity` recommended for 50+ entities.

## Privacy Layer

Core types in `privacy/` (no codegen needed). FilterFunc avoids circular deps:

```go
privacy.FilterFunc(func(ctx context.Context, f privacy.Filter) error {
    cc, ok := f.(interface{ HasColumn(string) bool })
    if !ok || !cc.HasColumn("workspace_id") {
        return privacy.Skip
    }
    f.WhereP(func(s *sql.Selector) {
        s.Where(sql.EQ(s.C("workspace_id"), workspaceID))
    })
    return privacy.Skip
})
```

Rules: `DenyIfNoViewer()`, `HasRole()`, `HasAnyRole()`, `IsOwner()`, `TenantRule()`, `AlwaysAllowRule()`, `AlwaysDenyRule()`, `And()`, `Or()`, `Not()`.

## Features System

```go
gen.WithFeatures(
    gen.FeaturePrivacy,            // Authorization policies
    gen.FeatureIntercept,          // Query interceptors
    gen.FeatureEntQL,              // Runtime query language
    gen.FeatureNamedEdges,         // Named edge loading
    gen.FeatureSnapshot,           // Schema snapshot
    gen.FeatureUpsert,             // ON CONFLICT support
    gen.FeatureGlobalID,           // Relay Global ID
    gen.FeatureAutoDefault,        // Auto DB defaults
    gen.FeatureWhereInputAll,      // All fields filterable (Ent-compat)
)
```

## Generated Code Structure

Uses entity sub-packages (not flat files) for compile isolation at 300+ schemas.

```
{target}/
├── velox.go                # Base types, Op, Value, type aliases
├── client.go               # Root Client, config, hooks/inters structs
├── errors.go               # Error types
├── tx.go                   # Transaction support
├── types.go                # Shared type helpers
├── runtime.go              # Version/Sum constants
├── entity/
│   ├── hooks.go            # HookStore, InterceptorStore (typed, shared pointer)
│   ├── velox.go            # Error aliases, aggregation helpers
│   └── {entity}.go         # Entity struct, Edges, builder interfaces
├── {entity}/               # Per-entity sub-package
│   ├── {entity}.go         # Table/column constants, ordering, load options
│   ├── where.go            # WHERE predicates (generic field descriptors)
│   ├── client.go           # Entity client (CRUD, Use, Hooks)
│   ├── create.go           # Create/CreateBulk builders
│   ├── update.go           # Update/UpdateOne builders
│   ├── delete.go           # Delete/DeleteOne builders
│   ├── mutation.go         # Mutation type
│   └── runtime.go          # init() — RegisterEntity + defaults/validators
├── query/
│   ├── helpers.go          # RegisterQueryFactory for all entities
│   └── {entity}.go         # Query builder + edge loading
├── predicate/predicate.go  # Predicate type aliases
├── hook/hook.go            # Hook type aliases
└── migrate/
    ├── migrate.go          # Migration runner
    └── schema.go           # Schema definitions for migration
```

```
       ↓ extends

                      GenWrapperCreate/Update/Delete, GenQueryPkg,

```

## Error Types

Sentinel: `ErrNotFound`, `ErrNotSingular`, `ErrTxStarted`.
Structured: `NotFoundError`, `NotSingularError`, `NotLoadedError`, `QueryError`, `MutationError`, `ValidationError`, `ConstraintError`, `PrivacyError`.
Checkers: `IsNotFound()`, `IsNotSingular()`, `IsConstraintError()`, `IsValidationError()`, etc.

## Supported Databases

PostgreSQL (primary), MySQL, SQLite. SQLite uses `modernc.org/sqlite` (pure Go). Driver name: `"sqlite"`. DSN: `_pragma=foreign_keys(1)`.
